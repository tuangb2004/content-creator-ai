MỞ ĐẦU

1. Lý do chọn đề tài

Trong bối cảnh cách mạng công nghệ 4.0 và sự phát triển mạnh mẽ của trí tuệ nhân tạo (AI), việc sáng tạo nội dung marketing đóng vai trò then chốt trong chiến lược kinh doanh của mọi doanh nghiệp. Theo thống kê từ Content Marketing Institute, các doanh nghiệp chi trung bình 25-30% ngân sách marketing cho việc sản xuất nội dung.

Tuy nhiên, quá trình sáng tạo nội dung truyền thống gặp nhiều thách thức:
- Tốn nhiều thời gian: Một bài blog chất lượng có thể mất 3-4 giờ để hoàn thành
- Chi phí nhân sự cao: Cần đội ngũ content writer, designer, video editor
- Khó duy trì tính nhất quán: Khi có nhiều người viết, giọng văn dễ bị khác nhau
- Áp lực deadline: Cần sản xuất nội dung liên tục cho nhiều nền tảng

Sự ra đời của AI Generative như ChatGPT, Gemini, Stable Diffusion đã mở ra kỷ nguyên mới cho sáng tạo nội dung. Tuy nhiên, các công cụ hiện có còn nhiều hạn chế:
- ChatGPT: Chỉ hỗ trợ văn bản, không tối ưu cho marketing chuyên sâu
- Jasper: Chi phí cao (~$49/tháng), không hỗ trợ tiếng Việt tốt
- Canva AI: Thiên về hình ảnh, hạn chế về văn bản chuyên nghiệp
- Các công cụ riêng lẻ: Phải sử dụng nhiều ứng dụng khác nhau

Từ những phân tích trên, em quyết định thực hiện đề tài "Xây dựng hệ thống tạo nội dung marketing sử dụng trí tuệ nhân tạo" với mục tiêu tạo ra một nền tảng tích hợp, dễ sử dụng và phù hợp với thị trường Việt Nam.

CHƯƠNG 1: CƠ SỞ LÝ THUYẾT

1.1. Tổng quan về đề tài

1.1.1. Giới thiệu lĩnh vực Content Marketing

Content Marketing (Tiếp thị nội dung) là chiến lược marketing tập trung vào việc tạo và phân phối nội dung có giá trị, phù hợp và nhất quán để thu hút và giữ chân đối tượng mục tiêu, từ đó thúc đẩy hành động có lợi cho doanh nghiệp.

Các loại nội dung marketing phổ biến:
- Blog posts và Articles: Bài viết dài, chuyên sâu về chủ đề
- Social Media Content: Caption, hashtags cho Instagram, Facebook, LinkedIn
- Video Content: Kịch bản cho Reels, TikTok, YouTube Shorts
- Email Marketing: Newsletter, cold emails, follow-up
- Visual Content: Banner, thumbnail, infographic

1.1.2. Định nghĩa các thuật ngữ

AI Generative (AI Tạo sinh): Là nhánh của trí tuệ nhân tạo có khả năng tạo ra nội dung mới như văn bản, hình ảnh, âm thanh dựa trên dữ liệu đã được huấn luyện.

Large Language Model (LLM): Mô hình ngôn ngữ lớn được huấn luyện trên hàng tỷ tham số, có khả năng hiểu và tạo văn bản giống con người.

Prompt Engineering: Kỹ thuật thiết kế và tối ưu câu lệnh để đạt được kết quả tốt nhất từ mô hình AI.

Serverless Architecture: Kiến trúc điện toán đám mây nơi nhà cung cấp quản lý hoàn toàn cơ sở hạ tầng, developer chỉ cần tập trung vào code.

1.2. Giới thiệu về AI Generative

1.2.1. Khái niệm

AI Generative là công nghệ sử dụng các mô hình học sâu (Deep Learning) để tạo ra nội dung mới. Khác với AI truyền thống chỉ phân tích dữ liệu, AI Generative có thể:
- Tạo văn bản: Viết bài, trả lời câu hỏi, dịch thuật
- Tạo hình ảnh: Vẽ tranh, thiết kế đồ họa từ mô tả
- Tạo code: Viết chương trình, debug lỗi
- Tạo âm thanh: Sáng tác nhạc, chuyển văn bản thành giọng nói

1.2.2. Các mô hình AI sử dụng trong dự án

Google Gemini:
- Gemini Pro: Mô hình xử lý văn bản với context window 32K tokens
- Gemini Pro Vision: Hỗ trợ tạo và phân tích hình ảnh
- Ưu điểm: Chất lượng cao, hỗ trợ tiếng Việt tốt
- Nhược điểm: Có chi phí, giới hạn tốc độ

Groq (Llama 3, Mixtral):
- Sử dụng LPU (Language Processing Unit) cho inference siêu nhanh
- Ưu điểm: Miễn phí tier cơ bản, tốc độ rất nhanh
- Nhược điểm: Chất lượng đầu ra không bằng Gemini

Pollinations AI:
- API miễn phí cho image generation
- Sử dụng Stable Diffusion models
- Ưu điểm: Hoàn toàn miễn phí
- Nhược điểm: Chất lượng hạn chế, không ổn định

1.3. Công nghệ Frontend - React.js

1.3.1. Giới thiệu React

React là thư viện JavaScript mã nguồn mở được phát triển bởi Meta (Facebook) vào năm 2013. React được sử dụng để xây dựng giao diện người dùng (UI) cho các ứng dụng web.

Đặc điểm chính:
- Component-based: Chia giao diện thành các thành phần độc lập, tái sử dụng
- Virtual DOM: Tối ưu hiệu suất bằng cách so sánh và cập nhật DOM ảo
- Declarative: Mô tả giao diện theo trạng thái, React tự động cập nhật
- Unidirectional Data Flow: Dữ liệu chảy một chiều từ parent xuống child

1.3.2. React Hooks

Hooks cho phép sử dụng state và các tính năng khác trong functional components:

useState: Quản lý state cục bộ
```javascript
const [count, setCount] = useState(0);
```

useEffect: Xử lý side effects (gọi API, subscription)
```javascript
useEffect(() => {
  // Code chạy sau khi render
  return () => {
    // Cleanup
  };
}, [dependencies]);
```

useContext: Truy cập dữ liệu toàn cục không cần prop drilling
```javascript
const value = useContext(MyContext);
```

1.3.3. Vite - Build Tool

Vite là build tool thế hệ mới với các ưu điểm:
- Hot Module Replacement (HMR) cực nhanh
- ES Modules native, không cần bundle khi dev
- Tối ưu hóa production build với Rollup

1.3.4. Tailwind CSS

Tailwind CSS là utility-first CSS framework:
- Không cần viết CSS riêng, dùng các class có sẵn
- Design system có thể tùy chỉnh qua config
- Responsive design với breakpoint prefixes (sm:, md:, lg:)
- Dark mode tích hợp sẵn

Ví dụ:
```html
<button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
  Button
</button>
```

1.4. Công nghệ Backend - Firebase

1.4.1. Giới thiệu Firebase

Firebase là nền tảng phát triển ứng dụng của Google, cung cấp các dịch vụ Backend-as-a-Service (BaaS):
- Authentication: Xác thực người dùng
- Cloud Firestore: Cơ sở dữ liệu NoSQL
- Cloud Functions: Serverless backend
- Cloud Storage: Lưu trữ file
- Hosting: Deploy static web

1.4.2. Firebase Authentication

Hỗ trợ nhiều phương thức xác thực:
- Email/Password: Đăng ký, đăng nhập với email
- OAuth Providers: Google, Facebook, Twitter, GitHub
- Custom Token: Xác thực từ backend khác (TikTok)
- Phone Authentication: OTP qua SMS

Tính năng bảo mật:
- Email Verification: Xác thực email người dùng
- Password Reset: Đặt lại mật khẩu qua email
- Multi-factor Authentication: Bảo mật 2 lớp

1.4.3. Cloud Firestore

Firestore là cơ sở dữ liệu NoSQL document-oriented:
- Real-time: Đồng bộ dữ liệu tức thời
- Offline Support: Hoạt động khi mất mạng
- Scalable: Tự động scale theo nhu cầu
- Security Rules: Bảo mật ở tầng database

Cấu trúc dữ liệu:
```
Collection → Documents → Fields
             └─ Subcollections
```

1.4.4. Cloud Functions

Firebase Cloud Functions cho phép chạy code backend:
- Trigger-based: Kích hoạt bởi HTTP request, database change, auth event
- Auto-scaling: Tự động mở rộng theo traffic
- Pay-per-execution: Chỉ trả tiền khi function chạy

Ví dụ:
```typescript
export const helloWorld = functions.https.onCall((data, context) => {
  return { message: "Hello World" };
});
```

1.5. Các AI Provider

1.5.1. Google Gemini API

Gemini là mô hình AI mới nhất của Google, thay thế Bard:
- Gemini Pro: Xử lý văn bản, context 32K tokens
- Gemini Pro Vision: Xử lý hình ảnh
- Gemini Ultra: Mô hình mạnh nhất (hạn chế access)

Tích hợp qua @google/generative-ai SDK:
```javascript
const genAI = new GoogleGenerativeAI(API_KEY);
const model = genAI.getGenerativeModel({ model: "gemini-pro" });
const result = await model.generateContent(prompt);
```

1.5.2. Groq API

Groq sử dụng Language Processing Unit (LPU) tự thiết kế:
- Inference speed: Nhanh gấp 10-100x GPU truyền thống
- Models: Llama 3 70B, Mixtral 8x7B
- Free tier: 30 requests/minute

1.5.3. Pollinations AI

API miễn phí cho image generation:
- Model: Stable Diffusion variants
- Endpoint đơn giản: GET request với prompt
- Không cần API key

1.6. Thanh toán trực tuyến - PayOS

1.6.1. Giới thiệu PayOS

PayOS là nền tảng thanh toán trực tuyến được phát triển bởi người Việt, tối ưu cho thị trường Việt Nam:
- Hỗ trợ 40+ ngân hàng Việt Nam
- QR Code thanh toán
- Webhook callback realtime
- API đơn giản, tài liệu tiếng Việt

1.6.2. Quy trình thanh toán

1. Client gọi API tạo payment link
2. Server tạo link với PayOS
3. User scan QR / chuyển khoản
4. PayOS gọi webhook thông báo
5. Server cập nhật trạng thái, cộng credits

```typescript
export const generateContent = functions.https.onCall(
  async (data, context) => {
    const userId = validateAuth(context);
    await checkRateLimit(userId);
    
    // Check credits
    const userDoc = await db.collection('users').doc(userId).get();
    const credits = userDoc.data()?.credits || 0;
    
    if (credits < requiredCredits) {
      throw new functions.https.HttpsError(
        'failed-precondition', 
        'Insufficient credits'
      );
    }
    
    // Generate content
    const result = await callAI(data.prompt, data.provider);
    
    // Deduct credits
    await userRef.update({ credits: credits - requiredCredits });
    
    return { content: result, creditsUsed: requiredCredits };
  }
);
```
